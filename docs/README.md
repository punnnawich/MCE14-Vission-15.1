# คู่มือการใช้งานระบบ MCE14 Vision
(3D Red Ball Tracking & Landing Prediction)

ระบบนี้ถูกพัฒนาเพื่อใช้กล้อง OAK-D Lite ตรวจจับลูกบอลสีแดงในรูปแบบ 3 มิติ (X, Y, Z) ร่วมกับโมเดล AI (YOLO) และใช้อัลกอริทึม **Kalman Filter** เข้ามาช่วยประเมินการเคลื่อนที่เพื่อพยากรณ์ **"พิกัดจุดตก" (Landing Point)** ของลูกบอล จากนั้นระบบจะยิงข้อมูลพิกัดนั้นด้วยความเร็วสูงผ่านโปรโตคอล UDP ไปยังไมโครคอนโทรลเลอร์ (ESP32) เพื่อนำไปสั่งการล้อหรือแขนกลให้เคลื่อนที่ไปรอรับลูกบอลต่อไป

---

## 📂 โครงสร้างและไฟล์ในโปรเจกต์

ในโฟลเดอร์รวบรวมโปรเจกต์นี้ มีไฟล์ที่ทำงานร่วมกันดังนี้:
1. `track_3d.py` : เป็นแกนหลัก (Main Program) เชื่อมต่อตากล้อง โมเดล YOLO และ Kalman Filter เข้าด้วยกัน
2. `udp_sender.py` : รับผิดชอบแพ็กข้อมูลเป็น Binary (ฐานสอง) และยิงข้อมูล UDP ออกไปยัง IP เป้าหมาย
3. `plot_3d.py` : (Optional) โปรแกรมตัวช่วยสำหรับเปิดกราฟ 3 มิติ จำลองและดูวิถีลูกบอลตกบนหน้าจอคอมฯ
4. `local_udp_receiver.py` : เป็นโปรแกรมตัวรับสมมติ สำหรับเทสการไหลของข้อมูลในเครื่องตัวเอง
5. `esp32_udp_receiver/esp32_udp_receiver.ino` : โค้ดฝั่งหุ่นยนต์ (Arduino IDE) สำหรับตัว ESP32 ไว้รับข้อมูลและคัดกรองแพ็กเกจ

## ⚙️ 2. การตั้งค่าระบบเครือข่าย WiFi & IP Address

หัวใจหลักของการส่งข้อมูลคือ คอมพิวเตอร์และ ESP32 **ต้องอยู่ในวง LAN หรือ WiFi เดียวกัน** แบ่งออกเป็น 2 วิธีหลัก (เลือกวิธีใดวิธีหนึ่ง):

### 🌐 วิธีที่ 1: ใช้คอมพิวเตอร์เป็นตัวปล่อยวง LAN (Mobile Hotspot) **-- วิธีที่แนะนำ! --**
ลดความหน่วงจากเร้าเตอร์บ้าน ทำให้สัญญาณยิงตรงถึง ESP32 ได้ไวที่สุด
1. เปิด **Mobile Hotspot** บนคอมพิวเตอร์ของคุณ (Share my internet connection)
2. ดูชื่อ **Network Name (SSID)** และ **Network Password** ของเน็ตคุณ
3. นำไปตั้งค่าในไฟล์ `esp32_udp_receiver.ino`:
   ```cpp
   const char* ssid = "ชื่อ_WIFI_HOTSPOT";      
   const char* password = "รหัสผ่าน_HOTSPOT"; 
   ```
4. กด Upload โค้ดลง ESP32
5. เปิดหน้าจอ **Serial Monitor** ใน Arduino IDE กดปุ่ม RST บนบอร์ดหนึ่งครั้ง และคัดลอกเลข **IP Address** ที่บอร์ดเชื่อมต่อสำเร็จ
6. นำเลข IP นำมาใส่ในไฟล์ตัวส่ง (Python): เข้าไปแก้ `udp_sender.py` หรือบรรทัดการส่งใน `track_3d.py` กรณีต้องการรวมตัว:
   ```python
   ESP32_IP = "IP_ADDRESS_ของ_ESP32" 
   ```

### 💻 วิธีที่ 2: การทดสอบในเครื่องตัวเอง (Local Test)
หากยังไม่ได้เสียบบอร์ด ESP32 และอยากทดสอบระบบจำลองก่อน
1. แก้ IP ในไฟล์ `udp_sender.py` ให้เป็น `ESP32_IP = "127.0.0.1"`
2. รันไฟล์ `python local_udp_receiver.py` ที่เครื่อง (มันจะทำตัวเป็น ESP32 ปลอมๆ คอยรับข้อมูล)
3. รันไฟล์ส่งข้อมูล เพื่อเช็คว่าพิกัดไหลลื่นดีหรือไม่

## 🧠 4. โครงสร้างข้อมูลที่ถูกส่งไปยัง ESP32 (Binary Packet)

เพื่อความเร็วสูงสุดจนไม่เกิดความหน่วง ข้อมูลถูกแปลงเป็น Binary แพ็กเกจขนาด **16 Bytes** ตรงๆ (ไม่ต้องเปลืองเวลาถอดรหัสข้อความ String) 
> ตัวแพ็กเกจจะถูกส่งเข้าที่ Port 12345 ของ ESP32

ประกอบด้วยข้อมูล 4 ส่วนย่อย (อย่างละ 4 Bytes):

1. **`Sequence Number (Uint32)`** 
   - ชุดตัวเลขที่รันขึ้นเรื่อยๆ (+1) ใช้ไว้กันปัญหา Packet หลุด หรือ Out-of-order 
   - ใน ESP32 มีโค้ดดักจับไว้แล้วว่า ถ้าเลขแพ็กเกจเก่าน้อยกว่ารอบปัจจุบัน จะทิ้งทันทีไม่เอามาประมวลผลให้หุ่นยนต์กระตุก
2. **`X_Land (Float)`** 
   - พิกัดแกน X (แนวนอนซ้ายขวา) ของจุดที่ลูกบอลจะตกลงพื้นดิน (หน่วยเป็นเซนติเมตร)
3. **`Y_Land (Float)`** 
   - พิกัดแกน Y (ความสูง) ถูกส่งมาเป็น `0.0` เสมอ เพราะโมเดลคณิตศาสตร์คำนวณจำลองการตกกระทบบนระนาบพื้น
4. **`Z_Land (Float)`** 
   - พิกัดแกน Z (ลึกเข้าไปด้านหน้าที่ตั้งกล้อง) ระยะที่จุดจะตกลงพื้น (หน่วยเป็นเซนติเมตร)

ESP32 ของคุณจะสามารถนำตัวแปร `x` กับ `z` ถ่ายทอดไปสั่งงานล้อ Mecanum หรือเขียนอัลกอริธึมหุ่นยนต์ให้หันหน้าพุ่งทะยานไปหาตำแหน่งนั้นได้เลย!
