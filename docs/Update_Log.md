## 🔄 รายการที่ทำการแก้ไขและอัปเกรด (Changelog)

### 1. 🌐 การจัดการ Error ของเครือข่าย UDP (Network Error Handling)
- **ไฟล์ที่แก้ไข:** `track_3d.py`
- **สิ่งที่ทำ:** เพิ่มระบบดักจับข้อผิดพลาด `try...except Exception as e` ในส่วนของการส่งข้อมูลผ่าน `sock.sendto` และ `esp32_sender.send_data_binary`
- **ผลลัพธ์:** หากไม่สามารถส่งข้อมูลไปที่ ESP32 ได้ (เช่น IP ผิด หรือสายหลุด) ระบบจะไม่พัง และจะแสดงข้อความสีแดงเตือนบนหน้าจอวิดีโอทันทีว่า `NET ERROR: [สาเหตุ]` พร้อมกับการพิมพ์ลงใน Terminal ทำให้แก้ปัญหาหน้างานได้ง่ายขึ้น

### 2. 🎯 การพัฒนาระบบคัดกรองเป้าหมาย (Target Selection Optimization)
- **ไฟล์ที่แก้ไข:** `track_3d.py`
- **สิ่งที่ทำ:** ปรับลูปการคัดเลือกวัตถุที่ตรวจจับได้จาก YOLO โดยแทนที่จะเลือกกล่องแรก ได้เพิ่มฟังก์ชัน `sorted()` เพื่อจัดเรียงกล่องตามค่าความมั่นใจ (Confidence Score) จากมากไปน้อย
- **ผลลัพธ์:** เมื่อพบวัตถุสีแดงหลายชิ้นในจอภาพเดียวกัน ระบบจะฉลาดพอที่จะเลือกติดตามเฉพาะชิ้นที่ "มีความมั่นใจที่จะเป็นลูกบอลเป้าหมายมากที่สุด" เพียงลูกเดียวเท่านั้น ช่วยลดปัญหา False Positive จากสิ่งรบกวนรอบนอก

### 3. ⚙️ การจัดระเบียบศูนย์กลางการตั้งค่า (Centralized Configurations)
- **ไฟล์ที่แก้ไข:** `track_3d.py`
- **สิ่งที่ทำ:** ดึงตัวแปรที่ถูกฝังไว้ (Hardcoded) กระจายตามส่วนต่างๆ ของโค้ดมารวมเป็นบล็อก `SYSTEM CONFIGURATIONS` ไว้ที่ด้านบนสุดของไฟล์ (บรรทัดที่ 90-100)
- **ผลลัพธ์:** ผู้ใช้งานสามารถปรับตั้งค่าระยะต่างๆ เช่น `CAMERA_HEIGHT_CM`, `ESP32_IP`, `UDP_PORT` และค่า `CONFIDENCE_THRESHOLD` ได้จากจุดเดียว ป้องกันความเสี่ยงในการแก้ไขโค้ดลึกๆ แล้วเกิดบัค

### 4. 🛡️ การกรองพิกัดสุดโต่ง (Outlier / Boundary Rejection)
- **ไฟล์ที่แก้ไข:** `track_3d.py`
- **สิ่งที่ทำ:** เพิ่มเงื่อนไขจำกัดขอบเขต `if -500 <= landing_pt[0] <= 500 and -1000 <= landing_pt[1] <= 1000:` ก่อนที่จะทำการแพ็กข้อมูลแบบ Binary เพื่อยิงไปหา ESP32
- **ผลลัพธ์:** หาก Kalman Filter ได้รับค่า Noise จากเงาสะท้อนใน Depth Map ที่ทำให้ผลลัพธ์พยากรณ์จุดตกไกลระดับหลายกิโลเมตร ระบบจะทำการตัดค่านั้นทิ้ง ไม่อนุญาตให้ส่งไปที่หุ่นยนต์ และจะขึ้นข้อความแจ้งเตือนบนหน้าจอว่า `OUT OF BOUNDS` ป้องกันอุบัติเหตุหุ่นยนต์กระชากหรือวิ่งชนขอบสนาม
